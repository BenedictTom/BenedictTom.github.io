1. ReentrantLockж•°жҚ®з»“жһ„дёҺз»§жүҝе…ізі»

1.1 ж•ҙдҪ“жһ¶жһ„

ReentrantLock еҶ…йғЁж ёеҝғе®һзҺ°дё»иҰҒйқ дёүдёӘеҶ…йғЁзұ»жқҘж”Ҝж’‘пјҡ

ж ёеҝғжҖқжғіпјҡReentrantLock еҸӘжҳҜдёҖдёӘеЈіеӯҗпјҢзңҹжӯЈзҡ„еҠ й”Ғ/и§Јй”ҒйҖ»иҫ‘йғҪе§”жүҳз»ҷдәҶ Sync дҪ“зі»гҖӮ

1.2 з»§жүҝе…ізі»еӣҫ

1.3 ж ёеҝғеҶ…йғЁзұ»иҜҰи§Ј

1.3.1 SyncпјҲжҠҪиұЎеҹәзұ»пјү

е…ій”®зү№зӮ№пјҡ

  • з»§жүҝдәҶ AQS (AbstractQueuedSynchronizer)
  • state иЎЁзӨәй”Ғзҡ„йҮҚе…Ҙж¬Ўж•°пјҲ0 = жІЎдәәжҢҒжңүпјҢ>0 = иў«жҹҗдёӘзәҝзЁӢжҢҒжңүпјҢеҖјдёәйҮҚе…Ҙж¬Ўж•°пјү
  • exclusiveOwnerThread и®°еҪ•жҢҒжңүй”Ғзҡ„зәҝзЁӢ
  • жҸҗдҫӣдәҶ acquire/release жЁЎжқҝпјҢе…·дҪ“з”ұеӯҗзұ»пјҲе…¬е№і/йқһе…¬е№іпјүе®һзҺ°иҺ·еҸ–йҖ»иҫ‘

1.3.2 NonfairSyncпјҲйқһе…¬е№ій”Ғе®һзҺ°пјү

дё»иҰҒзү№зӮ№пјҡжҸ’йҳҹжңәеҲ¶

  • зәҝзЁӢи°ғз”Ё lock() ж—¶пјҢе…Ҳе°қиҜ•з”Ё compareAndSetState(0, 1) жҠўй”ҒпјҲCASпјү
  • еҰӮжһңжҲҗеҠҹпјҢзӣҙжҺҘиҺ·еҫ—й”Ғ вҶ’ дёҚз®ЎйҳҹеҲ—йҮҢжҳҜеҗҰжңүдәәеңЁзӯү
  • еҰӮжһңеӨұиҙҘпјҢжүҚдјҡиө° AQS зҡ„йҳҹеҲ—жҢӮиө·йҖ»иҫ‘

дјҳзӮ№пјҡеҗһеҗҗйҮҸй«ҳпјҢеҮҸе°‘зәҝзЁӢеҲҮжҚў
зјәзӮ№пјҡеҸҜиғҪеҜјиҮҙйҘҘйҘҝпјҲдёҖзӣҙжңүж–°зәҝзЁӢжҸ’йҳҹпјҢйҳҹеҲ—йҮҢзҡ„иҖҒзәҝзЁӢиҝҹиҝҹжӢҝдёҚеҲ°й”Ғпјү

1.3.3 FairSyncпјҲе…¬е№ій”Ғе®һзҺ°пјү

дё»иҰҒзү№зӮ№пјҡдёҘж јжҺ’йҳҹ

  • жҜҸж¬ЎиҺ·еҸ–й”Ғж—¶пјҢе…ҲеҲӨж–ӯйҳҹеҲ—йҮҢжҳҜеҗҰжңүеүҚй©ұиҠӮзӮ№
  • еҰӮжһңжңүдәәеңЁжҺ’йҳҹ вҶ’ еҝ…йЎ»жҺ’йҳҹпјҢдёҚиғҪжҸ’йҳҹ
  • еҸӘжңүйҳҹеҲ—еүҚйқўжІЎдәәжҲ–иҖ…иҮӘе·ұеңЁйҳҹйҰ–пјҢжүҚиғҪе°қиҜ• CAS жҠўй”Ғ
  • hasQueuedPredecessors() е°ұжҳҜе…¬е№ізҡ„е…ій”® вҶ’ еҲӨж–ӯжҳҜеҗҰжңүзәҝзЁӢеңЁеүҚйқўжҺ’йҳҹ

дјҳзӮ№пјҡзәҝзЁӢиҺ·еҸ–й”Ғзҡ„йЎәеәҸе’ҢиҜ·жұӮйЎәеәҸдёҖиҮҙпјҢйҒҝе…ҚйҘҘйҘҝ
зјәзӮ№пјҡеҗһеҗҗйҮҸжҜ”йқһе…¬е№ій”ҒдҪҺ

2. AQSж ёеҝғжңәеҲ¶

2.1 AQSж ёеҝғеӯ—ж®ө

ж ёеҝғж•°жҚ®з»“жһ„пјҡ

  • stateпјҲintпјүпјҡиЎЁзӨәеҗҢжӯҘзҠ¶жҖҒпјҲеҰӮй”ҒжҳҜеҗҰиў«еҚ з”ЁпјҢйҮҚе…Ҙж¬Ўж•°пјҢдҝЎеҸ·йҮҸж•°йҮҸзӯүпјү
  • CLH йҳҹеҲ—пјҡдёҖдёӘ FIFO еҸҢеҗ‘йҳҹеҲ—пјҢз”ЁжқҘжҢӮиө·зӯүеҫ…зҡ„зәҝзЁӢпјҲheadгҖҒtail иҠӮзӮ№пјү

2.2 AQSж ёеҝғж–№жі•еҲҶзұ»

2.2.1 зӢ¬еҚ жЁЎејҸпјҲExclusiveпјү

рҹ‘ү з”ЁдәҺ дә’ж–Ҙй”ҒпјҲеҰӮ ReentrantLockпјү

ж–№жі• дҪңз”Ё е®һзҺ°ж–№
acquire(int arg) зӢ¬еҚ жЁЎејҸе…ҘеҸЈж–№жі• AQSе®һзҺ°
tryAcquire(int arg) зңҹжӯЈе°қиҜ•иҺ·еҸ–й”Ғ еӯҗзұ»е®һзҺ°
release(int arg) зӢ¬еҚ жЁЎејҸйҮҠж”ҫж–№жі• AQSе®һзҺ°
tryRelease(int arg) зңҹжӯЈйҮҠж”ҫй”Ғ еӯҗзұ»е®һзҺ°

2.2.2 е…ұдә«жЁЎејҸпјҲSharedпјү

рҹ‘ү з”ЁдәҺ е…Ғи®ёеӨҡдёӘзәҝзЁӢеҗҢж—¶иҺ·еҸ–зҡ„иө„жәҗпјҲеҰӮ SemaphoreгҖҒCountDownLatchпјү

ж–№жі• дҪңз”Ё е®һзҺ°ж–№
acquireShared(int arg) е…ұдә«жЁЎејҸе…ҘеҸЈ AQSе®һзҺ°
tryAcquireShared(int arg) е…ұдә«иө„жәҗиҺ·еҸ–йҖ»иҫ‘ еӯҗзұ»е®һзҺ°
releaseShared(int arg) йҮҠж”ҫе…ұдә«иө„жәҗ AQSе®һзҺ°
tryReleaseShared(int arg) йҮҠж”ҫе…ұдә«иө„жәҗйҖ»иҫ‘ еӯҗзұ»е®һзҺ°

2.2.3 йҳҹеҲ—з®ЎзҗҶпјҲж ёеҝғжңәеҲ¶пјү

ж–№жі• дҪңз”Ё
addWaiter(Node mode) жҠҠеҪ“еүҚзәҝзЁӢе°ҒиЈ…жҲҗиҠӮзӮ№пјҢеҠ е…ҘйҳҹеҲ—
acquireQueued(Node node, int arg) иҮӘж—Ӣе°қиҜ•иҺ·еҸ–й”ҒпјҢеҰӮжһңеӨұиҙҘеҲҷйҳ»еЎһ
unparkSuccessor(Node node) е”ӨйҶ’йҳҹеҲ—дёӯзҡ„дёӢдёҖдёӘзӯүеҫ…зәҝзЁӢ
hasQueuedPredecessors() еҲӨж–ӯеҪ“еүҚзәҝзЁӢжҳҜеҗҰжңүеүҚй©ұиҠӮзӮ№пјҲе…¬е№ій”Ғдҫқиө–пјү

2.3 ж ёеҝғи°ғз”Ёй“ҫжҖ»з»“

зӢ¬еҚ жЁЎејҸпјҲReentrantLock дёәдҫӢпјүпјҡ

е…ұдә«жЁЎејҸпјҲSemaphore дёәдҫӢпјүпјҡ

3. еҠ й”ҒжөҒзЁӢиҜҰи§Ј

3.1 е®Ңж•ҙеҠ й”ҒжөҒзЁӢ

3.1.1 жңҖеӨ–еұӮе…ҘеҸЈпјҡlock()

йҖ»иҫ‘пјҡ

  • е…Ҳеҝ«йҖҹе°қиҜ• initialTryLock()пјҲCAS жҠўдёҖдёӢпјү
  • еҰӮжһңеӨұиҙҘпјҢиө° е®Ңж•ҙзҡ„ AQS йҳҹеҲ—жҺ’йҳҹиҺ·еҸ– acquire(1)

3.1.2 еҝ«йҖҹе°қиҜ•пјҡinitialTryLock()

йҖ»иҫ‘еҲҶжһҗпјҡ

  • state == 0 вҶ’ й”Ғз©әй—І вҶ’ еҰӮжһңйҳҹеҲ—жІЎдәәзӯү вҶ’ CAS жҠҠ state ж”№жҲҗ 1 вҶ’ жҠўеҲ°й”Ғ
  • state > 0 дё” owner жҳҜиҮӘе·ұ вҶ’ ж”ҜжҢҒйҮҚе…ҘпјҲstate++пјү
  • еҗҰеҲҷиҝ”еӣһ false вҶ’ жҠўй”ҒеӨұиҙҘ

вҡЎ иҝҷдёӘж–№жі•еҜ№еә” д№җи§Ӯеҝ«йҖҹи·Ҝеҫ„пјҢеӨ§йғЁеҲҶж—¶еҖҷй”Ғз«һдәүдёҚжҝҖзғҲпјҢе®ғе°ұзӣҙжҺҘжҲҗеҠҹдәҶ

3.1.3 ж…ўи·Ҝеҫ„пјҡacquire(int arg)

йҖ»иҫ‘пјҡ

  • tryAcquire(arg)пјҡеҶҚе°қиҜ•дёҖж¬ЎиҺ·еҸ–й”Ғ
  • еҰӮжһңиҝҳеӨұиҙҘ вҶ’ иҝӣе…Ҙ acquire(...)пјҢеҚі е…ҘйҳҹгҖҒиҮӘж—ӢгҖҒжҢӮиө·

3.1.4 ж ёеҝғиҺ·еҸ–жөҒзЁӢпјҡacquire(...)

иҝҷжҳҜ AQS жңҖеӨҚжқӮзҡ„ең°ж–№пјҢж ёеҝғеҫӘзҺҜйҖ»иҫ‘пјҡ

3.2 еҠ й”ҒжөҒзЁӢеӣҫ

4. и§Јй”ҒжөҒзЁӢиҜҰи§Ј

4.1 е®Ңж•ҙи§Јй”ҒжөҒзЁӢ

4.1.1 ReentrantLock.unlock()

и°ғз”Ёзҡ„жҳҜ AQS зҡ„ release(1)пјҢеҸӮж•° 1 д»ЈиЎЁгҖҢйҮҠж”ҫдёҖдёӘиө„жәҗпјҲдёҖж¬ЎйҮҚе…ҘпјүгҖҚ

4.1.2 AQS.release(int arg)

йҖ»иҫ‘пјҡ

  • tryRelease(arg)пјҡзңҹжӯЈйҮҠж”ҫй”ҒпјҲеӯҗзұ»е®һзҺ°пјү
  • еҰӮжһңе®Ңе…ЁйҮҠж”ҫжҲҗеҠҹпјҲstate еҪ’йӣ¶пјү вҶ’ е°қиҜ•е”ӨйҶ’йҳҹеҲ—йҮҢзҡ„дёӢдёҖдёӘзәҝзЁӢ
  • иҝ”еӣһ true иЎЁзӨәй”Ғе·Із»Ҹз©әй—І

4.1.3 tryRelease(arg) пјҲReentrantLock зҡ„е®һзҺ°пјү

йҖ»иҫ‘еҲҶжһҗпјҡ

  • state - releasesпјҡеҮҸе°‘йҮҚе…Ҙи®Ўж•°
  • еҰӮжһң state == 0пјҡ
    • й”Ғе®Ңе…ЁйҮҠж”ҫ
    • жё…з©әжҢҒжңүзәҝзЁӢи®°еҪ•пјҲexclusiveOwnerThread = nullпјү
    • иҝ”еӣһ true
  • еҗҰеҲҷеҸӘжҳҜйғЁеҲҶйҮҠж”ҫпјҲжҜ”еҰӮйҮҚе…Ҙ 3 ж¬ЎпјҢйҮҠж”ҫдёҖж¬Ў вҶ’ state д»Һ 3 еҸҳ 2пјүпјҢиҝ”еӣһ false

4.1.4 е”ӨйҶ’еҗҺз»§зәҝзЁӢпјҡunparkSuccessor(Node h)

йҖ»иҫ‘пјҡ

  • е…ҲжҠҠ head иҠӮзӮ№зҡ„зҠ¶жҖҒ жё…йӣ¶
  • жүҫеҲ°з¬¬дёҖдёӘжңүж•Ҳзҡ„еҗҺз»§иҠӮзӮ№пјҲзӯүеҫ…зҠ¶жҖҒзҡ„иҠӮзӮ№пјү
  • и°ғз”Ё LockSupport.unpark(s.thread) е”ӨйҶ’е®ғ

5.3 йҖүжӢ©е»әи®®

  • й«ҳе№¶еҸ‘еңәжҷҜпјҡйҖүжӢ©йқһе…¬е№ій”ҒпјҢжҖ§иғҪжӣҙеҘҪ
  • йңҖиҰҒдёҘж јйЎәеәҸпјҡйҖүжӢ©е…¬е№ій”ҒпјҢйҒҝе…ҚйҘҘйҘҝ
  • дёҖиҲ¬жғ…еҶөпјҡй»ҳи®Өйқһе…¬е№ій”ҒеҚіеҸҜ

6. зӣёе…ійқўиҜ•йўҳ

6.1 AQSж ёеҝғеҺҹзҗҶ

Q: AQSзҡ„ж ёеҝғеҺҹзҗҶжҳҜд»Җд№Ҳпјҹ

A: AQSпјҲAbstractQueuedSynchronizerпјүжҳҜJavaе№¶еҸ‘еҢ…дёӯзҡ„ж ёеҝғеҗҢжӯҘеҷЁпјҢе…¶ж ёеҝғеҺҹзҗҶеҢ…жӢ¬пјҡ

  1. зҠ¶жҖҒз®ЎзҗҶпјҡйҖҡиҝҮvolatile int stateеӯ—ж®өз®ЎзҗҶеҗҢжӯҘзҠ¶жҖҒ
  2. йҳҹеҲ—жңәеҲ¶пјҡдҪҝз”ЁCLHйҳҹеҲ—пјҲFIFOеҸҢеҗ‘й“ҫиЎЁпјүз®ЎзҗҶзӯүеҫ…зәҝзЁӢ
  3. жЁЎжқҝж–№жі•жЁЎејҸпјҡжҸҗдҫӣacquire/releaseжЁЎжқҝпјҢеӯҗзұ»е®һзҺ°tryAcquire/tryRelease
  4. зӢ¬еҚ /е…ұдә«жЁЎејҸпјҡж”ҜжҢҒзӢ¬еҚ й”ҒпјҲеҰӮReentrantLockпјүе’Ңе…ұдә«й”ҒпјҲеҰӮSemaphoreпјү

ж ёеҝғжҖқжғіпјҡйҖҡиҝҮCASж“ҚдҪңз®ЎзҗҶstateзҠ¶жҖҒпјҢйҖҡиҝҮйҳҹеҲ—з®ЎзҗҶзӯүеҫ…зәҝзЁӢпјҢе®һзҺ°й«ҳж•Ҳзҡ„зәҝзЁӢеҗҢжӯҘгҖӮ

6.2 ReentrantLockе®һзҺ°еҺҹзҗҶ

Q: ReentrantLockжҳҜеҰӮдҪ•е®һзҺ°еҸҜйҮҚе…Ҙзҡ„пјҹ

A: ReentrantLockйҖҡиҝҮд»ҘдёӢжңәеҲ¶е®һзҺ°еҸҜйҮҚе…Ҙпјҡ

  1. stateи®Ўж•°пјҡжҜҸж¬ЎиҺ·еҸ–й”Ғж—¶state+1пјҢйҮҠж”ҫж—¶state-1
  2. ownerи®°еҪ•пјҡexclusiveOwnerThreadи®°еҪ•жҢҒжңүй”Ғзҡ„зәҝзЁӢ
  3. йҮҚе…ҘеҲӨж–ӯпјҡеңЁtryAcquireдёӯжЈҖжҹҘеҪ“еүҚзәҝзЁӢжҳҜеҗҰдёәowner
  4. и®Ўж•°з®ЎзҗҶпјҡйҮҚе…Ҙж—¶stateйҖ’еўһпјҢе®Ңе…ЁйҮҠж”ҫж—¶stateеҪ’йӣ¶

6.3 е…¬е№ій”Ғvsйқһе…¬е№ій”Ғ

Q: е…¬е№ій”Ғе’Ңйқһе…¬е№ій”Ғзҡ„еҢәеҲ«жҳҜд»Җд№Ҳпјҹ

A: дё»иҰҒеҢәеҲ«еҰӮдёӢпјҡ

  1. иҺ·еҸ–зӯ–з•Ҙпјҡ

    • йқһе…¬е№ій”ҒпјҡзӣҙжҺҘCASжҠўй”ҒпјҢеҸҜиғҪжҸ’йҳҹ
    • е…¬е№ій”ҒпјҡжЈҖжҹҘйҳҹеҲ—пјҢдёҘж јFIFO

  2. жҖ§иғҪе·®ејӮпјҡ

    • йқһе…¬е№ій”ҒпјҡеҗһеҗҗйҮҸй«ҳпјҢеҮҸе°‘зәҝзЁӢеҲҮжҚў
    • е…¬е№ій”ҒпјҡеҗһеҗҗйҮҸдҪҺпјҢдҝқиҜҒе…¬е№іжҖ§

  3. йҘҘйҘҝй—®йўҳпјҡ

    • йқһе…¬е№ій”ҒпјҡеҸҜиғҪеҜјиҮҙйҘҘйҘҝ
    • е…¬е№ій”ҒпјҡйҒҝе…ҚйҘҘйҘҝ

  4. е®һзҺ°еӨҚжқӮеәҰпјҡ

    • йқһе…¬е№ій”Ғпјҡе®һзҺ°з®ҖеҚ•
    • е…¬е№ій”ҒпјҡйңҖиҰҒhasQueuedPredecessors()жЈҖжҹҘ

6.4 AQSйҳҹеҲ—жңәеҲ¶

Q: AQSзҡ„CLHйҳҹеҲ—жҳҜеҰӮдҪ•е·ҘдҪңзҡ„пјҹ

A: CLHйҳҹеҲ—е·ҘдҪңжңәеҲ¶пјҡ

  1. иҠӮзӮ№з»“жһ„пјҡжҜҸдёӘзӯүеҫ…зәҝзЁӢе°ҒиЈ…дёәNodeиҠӮзӮ№
  2. е…Ҙйҳҹж“ҚдҪңпјҡйҖҡиҝҮCASж“ҚдҪңе°ҶиҠӮзӮ№ж·»еҠ еҲ°йҳҹе°ҫ
  3. еҮәйҳҹж“ҚдҪңпјҡиҺ·еҸ–й”Ғзҡ„зәҝзЁӢжҲҗдёәж–°зҡ„headиҠӮзӮ№
  4. е”ӨйҶ’жңәеҲ¶пјҡйҮҠж”ҫй”Ғж—¶е”ӨйҶ’дёӢдёҖдёӘзӯүеҫ…иҠӮзӮ№
  5. зҠ¶жҖҒз®ЎзҗҶпјҡйҖҡиҝҮNode.statusз®ЎзҗҶиҠӮзӮ№зҠ¶жҖҒ

ж ёеҝғжөҒзЁӢпјҡ

6.5 й”Ғзҡ„жҖ§иғҪдјҳеҢ–

Q: AQSеңЁжҖ§иғҪж–№йқўеҒҡдәҶе“ӘдәӣдјҳеҢ–пјҹ

A: дё»иҰҒдјҳеҢ–еҢ…жӢ¬пјҡ

  1. еҝ«йҖҹи·Ҝеҫ„пјҡinitialTryLock()зӣҙжҺҘCASпјҢйҒҝе…Қе…ҘйҳҹејҖй”Җ
  2. иҮӘж—Ӣзӯүеҫ…пјҡеңЁparkеүҚиҝӣиЎҢзҹӯжҡӮиҮӘж—ӢпјҢеҮҸе°‘дёҠдёӢж–ҮеҲҮжҚў
  3. CASж“ҚдҪңпјҡдҪҝз”ЁCASдҝқиҜҒеҺҹеӯҗжҖ§пјҢйҒҝе…ҚйҮҚйҮҸзә§й”Ғ
  4. йҳҹеҲ—дјҳеҢ–пјҡCLHйҳҹеҲ—еҮҸе°‘й”Ғз«һдәүпјҢжҸҗй«ҳе№¶еҸ‘жҖ§иғҪ
  5. зҠ¶жҖҒеҺӢзј©пјҡе°ҶеӨҡдёӘзҠ¶жҖҒдҝЎжҒҜеҺӢзј©еҲ°stateеӯ—ж®өдёӯ

6.6 жӯ»й”Ғйў„йҳІ

Q: еҰӮдҪ•йҒҝе…ҚдҪҝз”ЁReentrantLockж—¶зҡ„жӯ»й”Ғпјҹ

A: йҒҝе…Қжӯ»й”Ғзҡ„зӯ–з•Ҙпјҡ

  1. еӣәе®ҡйЎәеәҸпјҡжҢүеӣәе®ҡйЎәеәҸиҺ·еҸ–еӨҡдёӘй”Ғ
  2. и¶…ж—¶жңәеҲ¶пјҡдҪҝз”ЁtryLock(timeout)йҒҝе…Қж— йҷҗзӯүеҫ…
  3. й”Ғзҡ„зІ’еәҰпјҡе°ҪйҮҸеҮҸе°Ҹй”Ғзҡ„зІ’еәҰпјҢеҮҸе°‘жҢҒжңүж—¶й—ҙ
  4. йҒҝе…ҚеөҢеҘ—пјҡйҒҝе…ҚеңЁжҢҒжңүй”Ғж—¶иҺ·еҸ–е…¶д»–й”Ғ
  5. дҪҝз”Ёе·Ҙе…·пјҡдҪҝз”ЁReentrantReadWriteLockзӯүжӣҙз»ҶзІ’еәҰзҡ„й”Ғ

6.7 жәҗз Ғи®ҫи®ЎжЁЎејҸ

Q: AQSдҪҝз”ЁдәҶе“Әдәӣи®ҫи®ЎжЁЎејҸпјҹ

A: дё»иҰҒдҪҝз”ЁдәҶд»ҘдёӢи®ҫи®ЎжЁЎејҸпјҡ

  1. жЁЎжқҝж–№жі•жЁЎејҸпјҡacquire/releaseжҸҗдҫӣжЁЎжқҝпјҢtryAcquire/tryReleaseз”ұеӯҗзұ»е®һзҺ°
  2. зӯ–з•ҘжЁЎејҸпјҡйҖҡиҝҮдёҚеҗҢзҡ„Syncе®һзҺ°зұ»жҸҗдҫӣдёҚеҗҢзҡ„й”Ғзӯ–з•Ҙ
  3. зҠ¶жҖҒжЁЎејҸпјҡйҖҡиҝҮstateеӯ—ж®өз®ЎзҗҶдёҚеҗҢзҡ„еҗҢжӯҘзҠ¶жҖҒ
  4. и§ӮеҜҹиҖ…жЁЎејҸпјҡйҖҡиҝҮйҳҹеҲ—з®ЎзҗҶзӯүеҫ…зәҝзЁӢпјҢе®һзҺ°зәҝзЁӢй—ҙзҡ„йҖҡзҹҘжңәеҲ¶

6.8 е®һйҷ…еә”з”ЁеңәжҷҜ

Q: еңЁе®һйҷ…йЎ№зӣ®дёӯпјҢд»Җд№Ҳж—¶еҖҷйҖүжӢ©ReentrantLockиҖҢдёҚжҳҜsynchronizedпјҹ

A: йҖүжӢ©ReentrantLockзҡ„еңәжҷҜпјҡ

  1. йңҖиҰҒе…¬е№іжҖ§пјҡйңҖиҰҒдҝқиҜҒзәҝзЁӢиҺ·еҸ–й”Ғзҡ„йЎәеәҸ
  2. йңҖиҰҒи¶…ж—¶пјҡдҪҝз”ЁtryLock(timeout)йҒҝе…Қж— йҷҗзӯүеҫ…
  3. йңҖиҰҒдёӯж–ӯпјҡдҪҝз”ЁlockInterruptibly()е“Қеә”дёӯж–ӯ
  4. йңҖиҰҒеӨҡдёӘжқЎд»¶еҸҳйҮҸпјҡдҪҝз”ЁConditionе®һзҺ°еӨҚжқӮзҡ„зӯүеҫ…/йҖҡзҹҘ
  5. йңҖиҰҒеҸҜйҮҚе…ҘпјҡеҗҢдёҖдёӘзәҝзЁӢйңҖиҰҒеӨҡж¬ЎиҺ·еҸ–й”Ғ
  6. жҖ§иғҪиҰҒжұӮй«ҳпјҡеңЁй«ҳе№¶еҸ‘еңәжҷҜдёӢжҖ§иғҪжӣҙеҘҪ

жҖ»з»“

  1. зҠ¶жҖҒз®ЎзҗҶпјҡйҖҡиҝҮvolatile stateеӯ—ж®өз®ЎзҗҶеҗҢжӯҘзҠ¶жҖҒ
  2. йҳҹеҲ—жңәеҲ¶пјҡдҪҝз”ЁCLHйҳҹеҲ—з®ЎзҗҶзӯүеҫ…зәҝзЁӢ
  3. жЁЎжқҝж–№жі•пјҡжҸҗдҫӣз»ҹдёҖзҡ„еҗҢжӯҘжЎҶжһ¶
  4. жҖ§иғҪдјҳеҢ–пјҡеҝ«йҖҹи·Ҝеҫ„гҖҒиҮӘж—Ӣзӯүеҫ…зӯүдјҳеҢ–зӯ–з•Ҙ
