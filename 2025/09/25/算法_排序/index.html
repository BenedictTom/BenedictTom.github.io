算法_常见排序及复杂度比较

📊 常见排序算法复杂度一览表

排序算法 最好时间复杂度 平均时间复杂度 最坏时间复杂度 空间复杂度 是否稳定
冒泡排序 O(n) O(n²) O(n²) O(1) ✅ 是
选择排序 O(n²) O(n²) O(n²) O(1) ❌ 否
插入排序 O(n) O(n²) O(n²) O(1) ✅ 是
快速排序 O(n log n) O(n log n) O(n²) O(log n) ❌ 否
归并排序 O(n log n) O(n log n) O(n log n) O(n) ✅ 是
堆排序 O(n log n) O(n log n) O(n log n) O(1) ❌ 否
计数排序 O(n + k) O(n + k) O(n + k) O(k) ✅ 是
桶排序 O(n) O(n) O(n²) O(n + k) ✅ 是
基数排序 O(n × d) O(n × d) O(n × d) O(n + d) ✅ 是

🔹 n：数据量
🔹 k：数据范围（如 0~100）
🔹 d：数字的位数（如 3 位数）

🧩 逐个讲解（通俗版）

1. 冒泡排序（Bubble Sort）

  • 思想：像气泡一样，小的往上浮。两两比较，大的往后换。
  • 特点：
    • 简单易懂，但非常慢。
    • 如果数据几乎有序，它可以很快（O(n)）。
  • 适合：教学、数据极小、或已基本有序的情况。

💡 口诀：“慢得像蜗牛，只适合练手。”

2. 选择排序（Selection Sort）

  • 思想：每次从剩余元素中选最小的，放到前面。
  • 特点：
    • 每次都要扫一遍找最小，不管数据是否有序，都是 O(n²)。
    • 不稳定（相同值可能被打乱顺序）。
  • 优点：交换次数最少（最多 n-1 次）。

💡 口诀：“每次都选最小的，但太笨重。”

3. 插入排序（Insertion Sort）

  • 思想：像打扑克时整理牌——每拿到一张新牌，把它插到正确位置。
  • 特点：
    • 数据越有序越快，最好情况 O(n)。
    • 小数据量时效率不错，常用于优化快排（小数组切换成插入排序）。
  • 稳定，适合在线排序（边来边排）。

💡 口诀：“小数据之王，有序数据飞快。”

4. 快速排序（Quick Sort）

  • 思想：选一个“基准”（pivot），把小于它的放左边，大于的放右边，然后递归处理左右。
  • 特点：
    • 平均最快的通用排序算法，实际应用广泛（如 C++ 的 sort()）。
    • 最坏情况 O(n²)（比如已经有序时选了最差 pivot），但可以通过“随机化 pivot”避免。
    • 不稳定，但空间 O(log n)（递归栈深度）。
  • 缺点：最坏情况性能差，不适合对稳定性要求高的场景。

💡 口诀：“平均最快，但可能翻车。”

5. 归并排序（Merge Sort）

  • 思想：分而治之——不断二分，直到只剩一个元素，再两两合并成有序数组。
  • 特点：
    • 稳定，且最坏也是 O(n log n)，非常可靠。
    • 需要额外 O(n) 空间存储临时数组。
    • 适合链表排序、外部排序（大数据无法全加载进内存）。
  • Java 的 Arrays.sort() 对对象数组就用归并排序。

💡 口诀：“稳如老狗，空间换时间。”

6. 堆排序（Heap Sort）

  • 思想：利用“堆”这种数据结构（最大堆/最小堆），不断取出堆顶元素。
  • 特点：
    • 时间稳定 O(n log n)，空间 O(1)（原地排序）。
    • 但不稳定，且常数较大，实际比快排慢。
    • 适合对空间敏感、又不想有 O(n²) 风险的场景。

💡 口诀：“不占空间，但不太快也不稳定。”

7. 计数排序（Counting Sort）

  • 思想：不是比较，而是统计每个数出现的次数，然后按顺序输出。
  • 前提：数据范围小（比如 0~100 分的成绩）。
  • 特点：
    • 非比较排序，可以突破 O(n log n) 下限。
    • O(n + k)，当 k 不大时非常快。
    • 稳定，但需要额外 O(k) 空间。

💡 口诀：“适合分数、年龄这类小范围整数。”

8. 桶排序（Bucket Sort）

  • 思想：把数据分到多个“桶”里，每个桶内单独排序（可用插入排序），最后合并。
  • 前提：数据分布均匀。
  • 特点：
    • 最好 O(n)，但最坏 O(n²)（所有数据都进了同一个桶）。
    • 适合浮点数、均匀分布的数据。

💡 口诀：“数据均匀时最快，像快递分拣。”

9. 基数排序（Radix Sort）

  • 思想：按位排序（个位、十位、百位……），用计数排序作为子过程。
  • 前提：整数或字符串等可按位比较的数据。
  • 特点：
    • 不是基于比较的，所以可以 O(n × d)，d 是位数。
    • 稳定，适合固定长度的键排序（如手机号、身份证号）。

💡 口诀：“按位排队，适合固定格式数据。”

✅ 总结：怎么选？

场景 推荐算法
小数据（< 50） 插入排序
一般情况，追求速度 快速排序
要稳定、最坏也要快 归并排序
空间紧张，不能用额外内存 堆排序
数据范围小（如 0~100） 计数排序
浮点数、均匀分布 桶排序
整数、字符串、按位排序 基数排序

🎯 一句话总结：

  • O(n²)：冒泡、选择、插入 —— 小数据可用。
  • O(n log n)：快排、归并、堆排 —— 通用主力。
  • O(n)：计数、桶、基数 —— 特殊数据下的“外挂”。
