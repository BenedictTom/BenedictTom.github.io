栈

栈的特点

栈（Stack）是一种**后进先出（LIFO - Last In First Out）**的线性数据结构，具有以下特点：

  1. 操作限制：只能在栈顶进行插入和删除操作
  2. 后进先出：最后入栈的元素最先出栈
  3. 线性结构：元素之间存在一对一的关系
  4. 操作简单：主要操作只有入栈（push）和出栈（pop）

栈的基本操作

  • push() - 入栈，将元素添加到栈顶
  • pop() - 出栈，移除并返回栈顶元素
  • peek() - 查看栈顶元素，不删除
  • isEmpty() - 判断栈是否为空
  • size() - 获取栈中元素个数

队列

队列的特点

队列（Queue）是一种**先进先出（FIFO - First In First Out）**的线性数据结构，具有以下特点：

  1. 操作限制：只能在队尾插入，队首删除
  2. 先进先出：最先入队的元素最先出队
  3. 线性结构：元素之间存在一对一的关系
  4. 两端操作：支持在两端进行不同操作

队列的基本操作

  • offer() / add() - 入队，将元素添加到队尾
  • poll() / remove() - 出队，移除并返回队首元素
  • peek() / element() - 查看队首元素，不删除
  • isEmpty() - 判断队列是否为空
  • size() - 获取队列中元素个数

好的，我们来系统地梳理一下 Java 中常见的有界队列和无界队列。

一、核心概念

  • 有界队列 (Bounded Queue)：在创建时必须指定最大容量（capacity），队列满了之后，再尝试入队（put 或 offer）操作会：

    • 阻塞：等待队列有空位（如 BlockingQueue.put()）。
    • 失败：立即返回 false 或抛出异常（如 Queue.offer(e) 在某些实现中）。
    • 优点：可以防止内存无限增长，避免 OutOfMemoryError。
    • 缺点：生产者可能被阻塞，需要处理背压（backpressure）。

  • 无界队列 (Unbounded Queue)：没有预设的容量上限（或上限极大，如 Integer.MAX_VALUE）。

    • 理论上：可以无限添加元素。
    • 实际上：受限于 JVM 的内存大小。当内存耗尽时，会抛出 OutOfMemoryError。
    • 优点：生产者不会被阻塞，可以持续生产。
    • 缺点：如果消费者处理速度跟不上生产者，可能导致内存溢出。

二、常见队列分类

✅ 有界队列 (Bounded Queues)

这些队列在构造时必须指定容量，或者其内部结构决定了其有界性。

队列类 包 特点
ArrayBlockingQueue java.util.concurrent 基于数组的有界阻塞队列。使用单个锁（ReentrantLock）控制并发，支持公平/非公平策略。最典型的有界队列。
LinkedBlockingQueue java.util.concurrent 基于链表的可选有界队列。如果构造时指定了容量，则为有界；如果不指定，容量为 Integer.MAX_VALUE，此时行为上接近无界，但仍有上限。使用两个锁（读锁和写锁）实现读写分离，吞吐量通常高于 ArrayBlockingQueue。
SynchronousQueue java.util.concurrent 一个不存储元素的阻塞队列。每个 put 必须等待一个 take，反之亦然。可以看作是容量为 0 的队列。peek() 操作总是返回 null。
PriorityBlockingQueue java.util.concurrent 基于堆的可选有界优先级阻塞队列。如果不指定初始容量，它会自动扩容（类似 ArrayList），但可以通过构造函数指定初始容量。虽然可以扩容，但在特定配置下可视为有界。
DelayQueue java.util.concurrent 无界阻塞队列，但其内部使用的 PriorityQueue 在构造时可以指定初始容量。不过它本身通常被视为无界。

⚠️ 注意：LinkedBlockingQueue 和 PriorityBlockingQueue 是“可选有界”，但当指定了容量时，它们就是标准的有界队列。

✅ 无界队列 (Unbounded Queues)

这些队列在创建时不指定容量，或者其设计就是无界的。

队列类 包 特点
LinkedBlockingQueue java.util.concurrent 当不指定容量时，其容量为 Integer.MAX_VALUE（约 21 亿），在绝大多数应用场景下可视为无界。ThreadPoolExecutor 的 newFixedThreadPool 和 newSingleThreadExecutor 默认使用它。
ConcurrentLinkedQueue java.util.concurrent 基于无锁链表的线程安全无界队列。使用 CAS 操作保证线程安全，适用于高并发场景。非阻塞队列。
PriorityQueue java.util 基于堆的非线程安全优先级队列。内部数组会自动扩容，是无界的。
LinkedList java.util 基于双向链表的非线程安全队列。实现了 Queue 接口，可以作为无界队列使用。
ArrayDeque java.util 基于循环数组的非线程安全双端队列。虽然基于数组，但会自动扩容，是无界的。性能通常优于 LinkedList。

⚠️ 重要提示：即使是“无界”队列，也受内存限制。例如，向 ConcurrentLinkedQueue 中添加数亿个对象，最终会导致 OutOfMemoryError。

三、对比总结表

队列类 有界/无界 线程安全 阻塞 主要特点
ArrayBlockingQueue 有界 ✅ ✅ 数组实现，单锁，经典有界队列
LinkedBlockingQueue (指定容量) 有界 ✅ ✅ 链表实现，双锁，高吞吐
LinkedBlockingQueue (未指定容量) 无界 ✅ ✅ 容量为 Integer.MAX_VALUE
SynchronousQueue 有界 (容量=0) ✅ ✅ 不存储元素，直接传递
ConcurrentLinkedQueue 无界 ✅ ❌ 无锁，高并发，非阻塞
PriorityQueue 无界 ❌ ❌ 优先级队列，非线程安全
LinkedList 无界 ❌ ❌ 通用链表，可作队列
ArrayDeque 无界 ❌ ❌ 高性能双端队列

四、如何选择？

  • 需要控制资源、防止内存溢出 → 选择有界队列，如 ArrayBlockingQueue。
  • 高并发、低延迟、生产者不会太快 → 选择无界非阻塞队列，如 ConcurrentLinkedQueue。
  • 线程池工作队列：
    • newFixedThreadPool → LinkedBlockingQueue（无界）
    • newCachedThreadPool → SynchronousQueue（有界，容量=0）
  • 优先级处理 → PriorityBlockingQueue（可选有界）或 PriorityQueue（无界，非线程安全）。
  • 高性能栈/队列（单线程或外部同步）→ ArrayDeque（无界）。

五、关键结论

  1. 真正的“有界”：ArrayBlockingQueue、SynchronousQueue 是明确有界的。
  2. “可选有界”：LinkedBlockingQueue 和 PriorityBlockingQueue 可以配置为有界或无界。
  3. “理论无界”：ConcurrentLinkedQueue、PriorityQueue、LinkedList、ArrayDeque 等，虽然无界，但受内存限制。
  4. 无界不等于安全：使用无界队列时，必须确保消费者能跟上生产者的速度，否则迟早会 OOM。

总结

栈和队列是两种基础但非常重要的数据结构：

  1. 栈适用于需要”后进先出”的场景，如函数调用、表达式求值、括号匹配等
  2. 队列适用于需要”先进先出”的场景，如任务调度、消息传递、BFS遍历等
  3. Java提供了丰富的实现类，可以根据具体需求选择合适的实现
  4. 在实际开发中，栈和队列广泛应用于算法、并发编程、系统设计等各个领域
