Zset（有序集合）

什么是 Zset？

Zset（Sorted Set，有序集合）是 Redis 中一种非常强大且常用的数据结构。它像是一个“带分数的集合”——每个元素不仅唯一，还关联一个分数（score），Redis 会根据这个分数自动为元素排序。

你可以把它想象成一个班级的“成绩排行榜”：

  • 学生姓名 = 元素（member）
  • 考试分数 = 分数（score）
  • Redis 自动按分数从低到高或从高到低排序

通俗易懂的解释

想象你有一堆卡片，每张卡片上写着一个人的名字，并且贴了一个数字标签（比如跑步用时、积分、价格等）：

  • 所有卡片按数字标签从小到大整齐排列。
  • 同一个人只能有一张卡片（元素唯一）。
  • 数字可以重复，但名字不能重复。

当你想查“谁跑得最快”、“积分前10是谁”、“价格在100到500之间的商品有哪些”，Zset 就能飞快地告诉你答案。

核心特性

特性 说明
元素唯一 和 Set 一样，同一个 member 只能出现一次
分数可重复 不同 member 可以有相同 score
自动排序 按 score 升序排列，score 相同则按 member 字典序排
支持范围查询 可查 Top N、区间数据、排名等
高效查询 插入、删除、查询时间复杂度接近 O(log N)

底层使用 跳跃表（Skip List）+ 哈希表（Hash） 实现，兼顾排序与快速查找。

常用命令

1. ZADD key [NX\|XX] [CH] [INCR] score member [score member ...]

  • 作用：向有序集合添加一个或多个成员，或更新已存在成员的分数。

  • 示例：

    添加三个人的成绩。

  • 可选参数：

    • XX：只更新已存在的 member，不添加新成员
    • NX：只添加新 member，不更新已有
    • CH：返回发生变化的 member 数量（包括新增和分数变化）
    • INCR：对某个 member 的 score 做增量（类似 ZINCRBY）

2. ZSCORE key member

  • 作用：获取某个成员的分数。
  • 示例： 返回 95

3. ZRANK key member

  • 作用：获取 member 的排名（从小到大，从 0 开始）。
  • 示例：

ZREVRANK key member

  • 作用：从大到小排名（Top 排名，从 0 开始）。

4. ZRANGE key start stop [WITHSCORES]

  • 作用：按分数从小到大返回指定范围的成员。
  • 示例： 返回前三名（最低分开始）。

ZREVRANGE key start stop [WITHSCORES]

  • 作用：按分数从大到小返回指定范围的成员（常用作 Top N）。

5. ZRANGEBYSCORE key min max [WITHSCORES] [LIMIT offset count]

  • 作用：返回分数在 min 到 max 之间的所有成员。

  • 示例：

    返回分数在 90~100 之间的所有人。

  • 支持开区间：(90 表示大于 90

6. ZREM key member [member ...]

  • 作用：删除一个或多个成员。

7. ZCARD key

  • 作用：返回有序集合中元素的总数。

8. ZCOUNT key min max

  • 作用：统计分数在指定范围内的成员数量。

🌰 实际应用场景举例：游戏排行榜

✅ Zset 的经典使用场景

1. 排行榜系统

  • 游戏积分榜、直播热度榜、商品销量榜
  • 支持实时更新、Top N 查询

2. 优先级队列

  • 用 score 表示优先级或执行时间
  • 比如：延迟任务、定时提醒

3. 范围查询与统计

  • 价格区间筛选商品
  • 时间范围内访问记录（用时间戳做 score）

4. 去重 + 排序的数据集合

  • 既保证元素唯一，又能按权重排序
  • 比如：热搜词（热度值为 score）

5. 滑动窗口限流（配合 ZSET + 时间戳）

  • 记录用户请求时间，限制单位时间内的请求数
  • 比如：1 分钟内最多 100 次请求

🔔 注意事项

  • score 是双精度浮点数，范围约为 -9.2e63 到 +9.2e63
  • member 是字符串，最大 512MB
  • 内存占用较高：每个 member 和 score 都要存储，比 Hash 或 Set 略高
  • 避免大数据量频繁范围扫描：虽然高效，但全量遍历仍慢

✅ 总结：Zset 适合什么场景？

特点 是否适合 Zset
需要排序的数据 ✅ 核心优势
数据带权重/分数 ✅ 天然匹配
查询 Top N、Bottom N ✅ 极其高效
范围查询（如时间、价格） ✅ 支持良好
元素需要唯一性 ✅ 自动去重
实时性要求高 ✅ 写入查询都快

✅ 一句话总结：
只要你的数据有“排名”、“优先级”、“范围筛选”需求，Zset 往往就是 Redis 中最优雅、最高效的解决方案！
